# Análise de requisitos

Inicialmente, todo o levantamento de requisitos que o analista faz são apenas hipóteses, a confirmação de todas as informações se dará após a usabilidade da ferramenta.


## Tipos de requisitos

- **Requisito de Usuário:** o usuário explica o que ele precisa, de forma simples, é a origem do escopo;
- **Requisito de Negócio:** o que deseja que aconteça, ex: aumentar o faturamento em 10%;
- **Requisito de Sistema:** 
		* *requisitos funcionais:* funcionalidades que o usuário vai receber;
		* *requisitos não funcionais*: descrição da interface.
										

## Termos

Os termos corretos para descrever o que significa a análise de requisitos são: captar, desvendar e elicitar.

## O que é análise de requisitos

São atividades e técnicas que foram uma ligação entre as partes interessadas, com o intuito de compreender a estrutura e recomendar soluções para que a organização atinja suas metas.

Problemas + cenários possíveis = Objetivo

O analista de requisitos atua em 4 áreas da empresa:

 - planejamento estratégico;
 - definição de produtos;
 - modelagem e definição de processos;
 - análise de requisitos de software.


## Processo de elicitação de requisitos

Captar as necessidades do usuário:

 - Comunicação: tanto verbal como escrita.
 - Liderança: gerenciar as informações dentro do grupo que vou elicitar.

**Levantar evidências:**
1.1 Preparar a elicitação: Obter o entendimentos das atividades necessárias, definir pauta abordada, informar os envolvidos, garantir recursos (técnicos e de pessoal);
1.2 Conduzir a atividade de elicitação: envolver as pessoas;
1.3 Documentar os resultados: registrar de forma correta;
1.4 Confirmar os resultados: confirmar se as evidências documentadas estão de acordo, para deixas mais claro.

## Técnicas de levantamento de requisitos

*Observação:* verificar e ver todo o processo (politica e cultura organizacional), contexto ambiente.

*Visualizar:* tarefas e processos reais (implícito). 

*Questionário:* levar perguntas, validar afirmações e hipóteses.

 1. Montar uma sequência (ordem);
 2. Entrada (insumo) vs. saída;
 3. Quem são os envolvidos.

## Técnica de entrevista

Uma boa tática é gravar a entrevista. Você pode fazer em grupos ou entrevista individual. Na entrevista de grupos podem ocorrer alguns conflitos.

Tenha em mente:

 - Objetivo da reunião;
 - Que é realmente necessário;
 - Temo previsto;
 - A execução da reunião:
			 - deixar o entrevistado livre;
			 - questões pontuais;
			 - manter o foco.

## Desenho de interface

Desenhar a solução (tela, botão). Pedir para o usuário fazer o desenho.



## TOOLKIT

 **Ouvir o que não foi dito:**
  - Observação;
 - Desenho do processo;
 - Questionário;
 - Entrevista;
 - Desenho de telas;



**Resolução de conflitos:** 

 - Negociação: (ganha-ganha);
 - Participante externo;
 - Reforçar o objetivo da reunião;
 - Decisão (perde-ganha);
 - Gravação de áudio;
 - Votação.
> Conflitos: podem ser políticos, estruturais e comportamental.
> 
**Reunião planejada:**

 - Definir a pauta;
 - Pessoas essenciais na reunião;
 - Líder da reunião;
 - Local e recursos;
 - Uso do time-box.

## Principais características do analista de requisitos

 - Técnico: pensamento sistêmico / capacidade de abstração;
 - Comunicação: liderança / influência / negociação;
 - Inovação: coragem / evitar desperdício;
